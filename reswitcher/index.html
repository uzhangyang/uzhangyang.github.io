ReSwitcher: Automatically Refactoring Java Programs for Switch Expression

Several irregularities of the existing switch statement have long been an irritation to users. These include the default control flow behavior between switch labels (fall through), the default scoping in switch blocks (the whole block is treated as one scope), and the fact that switch works only as a statement, even though it is often more natural to express multi-way conditionals as expressions.

The current design of Java's switch statement follows closely languages such as C and C++, and supports fall through semantics by default. Whilst this traditional control flow is often useful for writing low-level code (such as parsers for binary encodings), as switch is used in higher-level contexts, its error-prone nature starts to outweigh its flexibility. The many break statements make the switch statement unnecessarily verbose, and this visual noise often masks hard to debug errors, where missing break statements would mean accidental fall through.

Starting from JDK 12, switch expression is introduced as the first preview feature. JDK 13 provides the second preview feature to support a new statement for yielding a value from a switch expression. Since JDK 14, this expression has become a standard feature, accompanied by a solution that using arrow labels to eliminate the need for break statements to remove the fall-through semantics. The coding way that simplifies everyday coding and keeps the code safe.

The C++ community recommends the fall-through attribute to comment when the program falls through. The default control flow of switch statements in Golang is to break out rather than to fall through. Switch expression has been proposed in C# and Java. A tool has been come up to convert switch statements to switch expressions automatically for C#. However, none of the existing automated refactoring tools supports refactoring Java programs for switch expression.

We presents a novel approach to automatically refactor Java programs for switch expression and develop a prototype ReSwitcher as an Eclipse plugin. The intention of ReSwitcher is to 1) remove the fall-through semantics from a switch statement, and 2) convert a switch statement into a switch expression.
